# 기능 점검 및 교사/학생 구분 정리

## 1. 교사(설정) vs 학생(확인) 구분

### 1.1 진입점

| 진입점 | 대상 | 비고 |
|--------|------|------|
| **랜딩** | 모두 | 로그인 없이 카드 클릭 가능. 내부에서 선생님/학생 로그인 유도 |
| **선생님 로그인** | 교사 | `isTeacherLoggedIn` → 대시보드·설정 권한 |
| **학생 로그인** | 학생 | 입장 코드로 참여 → `student` 설정, 게시글 작성·확인 등 |
| **선생님 대시보드** | 교사 전용 | 로그인 후만 진입. 학교 계획표·학급운영·게시판·할일·쪽지·계기교육·QR·AI문장·리포트 등 |

### 1.2 게시판별 쓰기 권한

| 게시판 ID | 교사 | 학생 | 비고 |
|-----------|------|------|------|
| **board** (자유게시판) | ✅ 쓰기·승인·설정 | ✅ 쓰기 (승인 후 공개) | ClassBoardApp |
| **gallery** (작품 갤러리) | ✅ 쓰기·승인·설정 | ✅ 쓰기 (승인 후 공개) | ClassBoardApp |
| **album** (우리반 사진첩) | ✅ 업로드 | ❌ 불가 | 선생님만 업로드 (BoardApp 로직) |
| **notice_board** (공지사항) | ✅ 작성 | ❌ 불가 | `allowStudentPost={false}` (ManagementApp) |
| **schoolplan** (학교 계획표) | ✅ 작성 | ❌ 불가 | PlannerApp 내 BoardApp, boardId 기준으로 글쓰기 버튼 비표시 |
| **writing, reading, math, learning** | ✅ 쓰기·승인 | ✅ 쓰기 (승인 후 공개) | LearningApp 등 |

### 1.3 기능별 교사 전용 여부

- **교사만 설정/관리**: 알림장 작성, 1인1역 배정, 포인트 지급, 쿠폰 발급, 학급회의 진행, 설문 생성·관리, 게시판 설정·승인, OMR 과제 출제, 만화 과제 생성·승인, 타이머 제어(교실 모드), 쪽지함 관리, 교무수첩(명렬표·상담·회의록), 학생 활동 리포트.
- **학생은 확인/참여**: 알림장·공지 열람, 1인1역·포인트·쿠폰 확인, 회의·설문 참여, 게시글 작성(승인 후 공개), OMR 제출, 만화 제출, 타이머 사용(개인), 쪽지 보내기·받기, 할일·스케줄 확인.

### 1.4 섞임 방지 정리

- **공지사항**: ManagementApp → `notice_board`에 `allowStudentPost={false}` 적용됨. 학생은 읽기만 가능.
- **우리반 사진첩**: ClassBoardApp에서 `boardId="album"`일 때 `allowStudentPost={false}` 전달. 선생님만 글쓰기 버튼 노출·업로드 가능.
- **학교 계획표**: PlannerApp의 BoardApp `boardId="schoolplan"`에서 글쓰기 버튼은 `boardId !== 'schoolplan'`일 때만 학생에게 노출되도록 되어 있음 → 학생은 쓰기 불가.
- **선생님 대시보드**: `currentApp === 'dashboard'`일 때만 TeacherDashboard 렌더링되며, 이 경로는 선생님 로그인 후 `onOpenDashboard`로만 진입.

## 2. 버튼/기능 점검 및 수정 사항

### 2.1 수정 완료

- **계기교육 퀴즈 선택지**: 퀴즈 발표 시 선택지 버튼이 클릭되어도 반응이 없었음. 선택지 클릭 시 해당 문제의 정답 노출(`setQuizReveal(currentQuizIdx)`)되도록 수정함.

### 2.2 동작 방식 정리

- **문장 다듬기, 빙고/단어찾기 AI**: OpenAI API 키가 있을 때만 동작. 없으면 안내 메시지 또는 빈 결과.
- **학생 입장 코드·새 코드 발급**: 선생님 프로필에 학교/학급이 연결된 경우에만 표시·발급 가능. "나의 정보 수정"으로 저장 후 갱신됨.
- **게시판 승인**: 선생님만 `PostCard`에서 승인/반려 가능. 학생은 본인 글만 대기 중으로 표시.

## 3. 비용 폭탄 주의

### 3.1 이미지

- **일일 업로드 한도**: `DAILY_IMAGE_LIMIT = 10` (로컬 쿼터 `localQuota.ts`). 같은 기기에서 하루 10장 초과 시 업로드 차단.
- **Supabase Storage**: 업로드 시 Storage 사용. 프로젝트별 용량·과금 정책 확인 필요.
- **이미지 보존 기간**: `IMAGE_RETENTION_DAYS = 90` 등 설정 확인 권장.

### 3.2 OpenAI (gpt-4o-mini)

- **호출 위치**: 문장 다듬기(1회/요청), 빙고 단어/단어쌍 생성(1회/요청), 단어찾기 단어 생성(1회/요청), 만화 장면 키워드(1회/장면), 리포트 서술 다듬기(1회/리포트), 푸터 뉴스(1회/로드). **자동 반복 호출·무한 루프 없음.**
- **권장**: OpenAI 대시보드에서 사용량·예산 알림 설정. 프로덕션에서는 API 키를 백엔드에서만 사용하도록 이전 권장.

### 3.3 Supabase

- **인증·DB·Storage**: 무료 티어 한도 확인. RLS로 불필요한 읽기/쓰기 제한 권장.

## 4. 보안 점검 요약

### 4.1 적용된 사항

- **역할 구분**: `isTeacherMode`(선생님 로그인)와 `student`(학생 입장)로 UI·쓰기 권한 분리. 게시판별로 `allowStudentPost`, `boardId === 'schoolplan'`, `boardId === 'album'` 등으로 학생 쓰기 제한.
- **비속어 필터**: `boardService`에서 `BANNED_WORDS`로 필터링 후 저장.
- **세션**: 선생님은 Supabase Auth, 학생은 입장 코드 기반 참여자 세션. `.env.local` 미커밋으로 키 노출 방지.

### 4.2 권장 사항

- **Supabase RLS**: 모든 테이블에 RLS 적용 및 정책 검토(선생님 본인 학급, 학생 본인 데이터만 접근).
- **입력값**: 게시글·알림·설문 등 사용자 입력은 저장·표시 시 이스케이프(React 기본 이스케이프 유지, 필요 시 sanitize).
- **OpenAI**: 프로덕션에서 API 키를 클라이언트에 두지 말고, 백엔드/서버리스에서만 호출.
